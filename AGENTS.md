# Инструкции для AI-агента (hdl-modules)

Краткий контекст проекта и единый стиль доработок. Подробная пошаговая инструкция для людей: [docs/ADDING_MODULES.md](docs/ADDING_MODULES.md).

## Назначение репозитория

Библиотека Verilog-модулей для FPGA-синтезатора (VitaSound). Модули тестируются в **Icarus Verilog**, waveform-картинки и README генерируются автоматически.

**Не путать:**

- `verilator_tests/` — legacy (Verilator + C++, keyboard/MIDI → soundcard/wav). В `make all` не входит.
- `hdl-modules-tester/` — UDP engine для VST (pull-only, только сеть). В `make all` не входит.

## Источник правды

| Что | Где |
|-----|-----|
| Список модулей, тесты, пути к PNG | [modules.yaml](modules.yaml) |
| Запуск симуляций | [tools/run_tests.py](tools/run_tests.py) |
| PNG из GTKWave | [tools/render_images.py](tools/render_images.py) |

| CI | [.github/workflows/test-and-docs.yml](.github/workflows/test-and-docs.yml) |

## Команды

```bash
make list              # id всех модулей
make sim ID=<id>       # симуляция одного модуля
make wave ID=<id>      # GTKWave для настройки test.gtkw
make test              # все симуляции
make images ID=<id>    # PNG (нужен out.vcd)
make docs              # README из modules.yaml
make all               # test → images → docs
```

## Стиль и ограничения

1. **Минимальный diff** — не трогать несвязанные модули и sandbox.
2. **Не редактировать README вручную** — файлы с `<!-- generated by tools/gen_readme.py -->` перегенерировать через `make docs`. Менять текст в `modules.yaml` или шаблонах `tools/templates/`.
3. **PNG всегда** `{test_dir}/test.png` (не в корне пакета).
4. **test.sh** — только тонкая обёртка на `tools/run_tests.py --id <id>`.
5. **Не коммитить** артефакты: `out.vcd`, `testbench`, `wave.ps`, `verilator_tests/obj_dir/`, `hdl-modules-tester/obj_dir/`.
6. **Коммитить** после `make all`: `test.png`, `test.gtkw`, `modules.yaml`, сгенерированные README.
7. RTL: стиль как в соседних `.v` (параметры, простые имена портов, комментарии только для неочевидного).
8. Testbench: `$dumpfile("out.vcd")`, `$dumpvars(0, testbench)`, `#TEST_DURATION` + `$finish`.

## Структура каталогов

**Common** (утилиты):

```
common/
  mymodule.v
  mymodule_test/          # или {name}_test — как у существующих
    testbench.v
    test.sh
    test.gtkw
    test.png
```

**Generation** (пакеты синтеза):

```
mypackage/
  *.v
  test/
    testbench.v
    test.sh
    test.gtkw
    test.png
  README.md               # генерируется
```

## Как добавить ещё один модуль (чеклист агента)

Ответ на вопрос «как добавить модуль» — выполнить по порядку:

### 1. RTL + testbench

- Создать `.v` и `test/testbench.v` (или `common/foo_test/`).
- В `sources` в yaml перечислить **все** `.v`, нужные для `iverilog` (включая зависимости, см. `vca`, `adsr`).

### 2. test.sh

```bash
#!/bin/bash
set -euo pipefail
ROOT="$(cd "$(dirname "$0")/../.." && pwd)"   # для generation/test/
# ROOT="$(cd "$(dirname "$0")/../../.." && pwd)"  # если глубже — считать от test_dir
exec python3 "$ROOT/tools/run_tests.py" --id <id>
```

Путь `ROOT`: от `test_dir` до корня репо (у `dds/test` — два уровня вверх, у `common/foo_test` — два уровня).

### 3. Проверить симуляцию

```bash
make sim ID=<id>
```

### 4. Настроить test.gtkw (вручную пользователем или через инструкции)

```bash
make wave ID=<id>
```

Сохранить zoom, список сигналов, **Analog Interpolated** для огибающих/шин. Без `test.gtkw` картинка будет хуже.

### 5. Запись в modules.yaml

- **common** → `categories[id=common].modules[]`
- **generation** → `categories[id=generation].packages[]`
- Обязательные поля: `id`, `description_ru`, `test_dir`, `sources`, `image`, `gtkw`
- Для generation: `readme`, опционально `title`, `variants[]`

### 6. Артефакты и CI

```bash
make test ID=<id>
make images ID=<id>
make docs          # обновляет все README репозитория
make all           # перед коммитом — полный прогон всех модулей
```

GitHub Actions проверяет `git diff` только для `README.md` и `modules.yaml`. PNG могут отличаться по байтам между локальной машиной и CI (разный GTKWave/шрифты) — их коммитят после локального `make images`.

### 7. Коммит

Включить: RTL, testbench, test.sh, test.gtkw, test.png, modules.yaml, README.

## Рендер PNG (не ломать)

- Используется **Grab_To_File**, не PostScript.
- При наличии `test.gtkw` **не** вызывать `Zoom Full` — масштаб и analog-формат из save-файла.
- `tools/gtkwaverc.render` игнорирует старый размер/позицию окна из gtkw (`ignore_savefile_size/pos/pane_pos`).

## Частые ошибки

| Проблема | Решение |
|----------|---------|
| Сплошные блоки на PNG | Уменьшить zoom в GTKWave, сохранить test.gtkw |
| Артефакты / дубли UI | Не менять логику ignore_* в gtkwaverc без причины |
| README не обновился | `make docs`, не править README руками |
| CI red на git diff README | `make docs`, закоммитить README и `modules.yaml` |
| CI red только на PNG | Нормально: PNG не сравниваются в CI; обновить локально `make images` |
| Новый id не находится | Добавить в modules.yaml, проверить `make list` |
